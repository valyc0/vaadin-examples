Build di produzione Vaadin 24 offline
Perché succede
Modalità	Cosa fa	Serve Node.js?	Serve Internet?
Dev (spring-boot:run)	Usa il dev-bundle precompilato incluso nei JAR Vaadin	❌ No	❌ No
Production (-Pproduction)	Fa un build frontend completo (scarica Node.js → npm install → Vite build)	✅ Sì	✅ Sì
Ecco perché in dev funziona e in produzione no: il build di produzione ha bisogno di Node.js e dei pacchetti npm, che tenta di scaricare da internet.

Soluzione passo-passo
L'idea: prepari Node.js + node_modules su una macchina con internet (non serve il TUO codice) e li porti sulla macchina offline.

FASE 1 — Su una macchina CON internet
1A. Scopri la versione di Node richiesta
Guarda il log del build che fallisce. Vedrai qualcosa come:

text

Installing Node version v18.18.0
Oppure cerca nel pom.xml del plugin vaadin-maven-plugin la proprietà nodeVersion.

1B. Scarica l'archivio Node.js
Da https://nodejs.org/dist/v18.18.0/ scarica l'archivio per il tuo OS:

text

# Linux
node-v18.18.0-linux-x64.tar.gz

# Windows
node-v18.18.0-win-x64.zip
1C. Genera node_modules
Crea un progetto Vaadin identico da start.vaadin.com con le stesse dipendenze Maven (stessa versione Vaadin, stessi add-on):

Bash

cd progetto-dummy
mvn clean package -Pproduction
A build completato, avrai nella root del progetto:

text

progetto-dummy/
├── node/                  ← Node.js scaricato da Vaadin
├── node_modules/          ← pacchetti npm
├── package.json
├── package-lock.json      ← (o pnpm-lock.yaml)
├── vite.config.ts
├── vite.generated.ts
└── ...
1D. Raccogli i file da trasferire
Copia su USB/rete interna:

text

node/                      (intera cartella)
node_modules/              (intera cartella)  
package.json
package-lock.json
FASE 2 — Sulla macchina OFFLINE
2A. Posiziona i file nel progetto
text

tuo-progetto/
├── src/
├── pom.xml
├── node/                  ← COPIATO QUI
├── node_modules/          ← COPIATO QUI
├── package.json           ← COPIATO QUI (verrà rigenerato, ma serve come base)
└── package-lock.json      ← COPIATO QUI
2B. Crea il file .npmrc nella root del progetto
properties

offline=true
prefer-offline=true
Questo dice a npm di non contattare il registry e usare solo ciò che è già in node_modules.

2C. Configura pom.xml
Nel vaadin-maven-plugin, aggiungi queste configurazioni:

XML

<build>
    <plugins>
        <plugin>
            <groupId>com.vaadin</groupId>
            <artifactId>vaadin-maven-plugin</artifactId>
            <version>${vaadin.version}</version>
            <configuration>
                <!-- NON riscaricare/aggiornare Node -->
                <nodeAutoUpdate>false</nodeAutoUpdate>
                
                <!-- Disabilita pnpm, usa npm (più semplice offline) -->
                <pnpmEnable>false</pnpmEnable>
                
                <!-- Usa ci build per rispettare il lock file -->
                <ciBuild>false</ciBuild>
            </configuration>
            <executions>
                <execution>
                    <goals>
                        <goal>prepare-frontend</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>
Nel profilo production assicurati che la stessa configurazione sia coerente:

XML

<profile>
    <id>production</id>
    <build>
        <plugins>
            <plugin>
                <groupId>com.vaadin</groupId>

Bash

mvn clean package -Pproduction

1️⃣ Filter HTTP in ingresso (già ce l’hai)

Nel tuo OncePerRequestFilter:

@Override
protected void doFilterInternal(HttpServletRequest request,
                                HttpServletResponse response,
                                FilterChain filterChain)
        throws ServletException, IOException {

    String correlationId = UUID.randomUUID().toString();
    MDC.put("correlationId", correlationId);
    MDC.put("clientIp", request.getRemoteAddr());
    MDC.put("incomingUrl", request.getRequestURI());

    Authentication auth = SecurityContextHolder.getContext().getAuthentication();
    if (auth != null) {
        MDC.put("user", auth.getName());
    }

    try {
        filterChain.doFilter(request, response);
    } finally {
        MDC.clear();
    }
}

👉 Qui NON devi loggare le chiamate esterne.
👉 Qui prepari solo il contesto.

2️⃣ Intercettare TUTTE le chiamate RestClient (parte fondamentale)

In Spring Boot 3 il RestClient usa sotto ClientHttpRequestInterceptor.

Crei un interceptor:

@Component
public class OutboundLoggingInterceptor implements ClientHttpRequestInterceptor {

    private static final Logger logger = LoggerFactory.getLogger(OutboundLoggingInterceptor.class);

    @Override
    public ClientHttpResponse intercept(HttpRequest request,
                                        byte[] body,
                                        ClientHttpRequestExecution execution) throws IOException {

        long start = System.currentTimeMillis();

        String clientIp = MDC.get("clientIp");
        String user = MDC.get("user");
        String incomingUrl = MDC.get("incomingUrl");
        String correlationId = MDC.get("correlationId");

        logger.info("OUTBOUND REQUEST | corrId={} | user={} | ip={} | from={} | to={} | method={} | body={}",
                correlationId,
                user,
                clientIp,
                incomingUrl,
                request.getURI(),
                request.getMethod(),
                new String(body, StandardCharsets.UTF_8));

        ClientHttpResponse response = execution.execute(request, body);

        long duration = System.currentTimeMillis() - start;

        logger.info("OUTBOUND RESPONSE | corrId={} | status={} | duration={}ms",
                correlationId,
                response.getStatusCode(),
                duration);

        return response;
    }
}
3️⃣ Registrare l’interceptor nel RestClient
@Bean
public RestClient restClient(OutboundLoggingInterceptor interceptor) {
    return RestClient.builder()
            .requestInterceptor(interceptor)
            .build();
}

🥇 SOLUZIONE 1 (più semplice): AsyncAppender + DBAppender

    <appender name="AUDIT_DB" class="ch.qos.logback.classic.db.DBAppender">

        </connectionSource>

    <!-- Async wrapper -->
    <appender name="ASYNC_AUDIT_DB" class="ch.qos.logback.classic.AsyncAppender">

        <discardingThreshold>0</discardingThreshold>
        <appender-ref ref="AUDIT_DB"/>
    </appender>

    <logger name="AUDIT_OUTBOUND" level="INFO" additivity="false">
        <appender-ref ref="ASYNC_AUDIT_DB"/>

@Service
public class MyService {

    private final RestClient client;

    public MyService(RestClient.Builder builder) {
        this.client = builder.baseUrl("http://localhost:8080").build();
    }

    public Page<ActionableContentDto<ContentDto>> getPage(int page, int size) {
        ParameterizedTypeReference<Page<ActionableContentDto<ContentDto>>> type =
            new ParameterizedTypeReference<>() {};

        return client.get()
                     .uri(uriBuilder -> uriBuilder
                             .path("/items")
                             .queryParam("page", page)
                             .queryParam("size", size)
                             .build())
                     .retrieve()
                     .body(type);
    }
}

// ===================== UI / View =====================

@Route("content")
public class ContentView extends VerticalLayout {

    private final MyService service;

    private Grid<ActionableContentDto<ContentDto>> grid;
    private Binder<ContentDto> binder;
    private FormLayout form;
    private TextField nameField;
    private TextField descriptionField;
    private TextField storagePathField;
    private Button saveButton;

    public ContentView(MyService service) {
        this.service = service;

        createGrid();
        createForm();

        add(grid, form);

        loadPage(0, 20);
    }

    private void createGrid() {
        grid = new Grid<>();
        grid.addColumn(wrapper -> wrapper.getContent().getName())
            .setHeader("Name");
        grid.addColumn(wrapper -> wrapper.getContent().getDescription())
            .setHeader("Description");

        // Colonna azioni con pulsanti condizionali
        grid.addColumn(new ComponentRenderer<>(wrapper -> {
            HorizontalLayout layout = new HorizontalLayout();

            if (wrapper.getActions().contains("DOWNLOAD")) {
                Button download = new Button("Download", e -> downloadFile(wrapper.getContent()));
                layout.add(download);
            }
            if (wrapper.getActions().contains("EDIT")) {
                Button edit = new Button("Edit", e -> editRow(wrapper));
                layout.add(edit);
            }
            return layout;
        })).setHeader("Actions");

        grid.setPageSize(20);
    }

    private void createForm() {
        binder = new Binder<>(ContentDto.class);
        form = new FormLayout();

        nameField = new TextField("Name");
        descriptionField = new TextField("Description");
        storagePathField = new TextField("Storage Path");

        binder.bind(nameField, "name");
        binder.bind(descriptionField, "description");
        binder.bind(storagePathField, "storage.path");

        saveButton = new Button("Save", e -> save());
        form.add(nameField, descriptionField, storagePathField, saveButton);
    }

    private void loadPage(int page, int size) {
        Page<ActionableContentDto<ContentDto>> p = service.getPage(page, size);
        grid.setItems(p.getContent());
    }

    private void editRow(ActionableContentDto<ContentDto> wrapper) {
        ContentDto content = wrapper.getContent();

        // Setta il bean del Binder
        binder.setBean(content);

        // Mostra/nascondi campi in base alle actions
        nameField.setReadOnly(!wrapper.getActions().contains("EDIT_NAME"));
        storagePathField.setVisible(wrapper.getActions().contains("VIEW_STORAGE"));
    }

    private void save() {
        ContentDto content = binder.getBean();
        // Logica per salvare il contentDto, ad esempio tramite service
        Notification.show("Saved: " + content.getName());
    }

    private void downloadFile(ContentDto content) {
        // Logica download
        Notification.show("Download: " + content.getName());
    }
}
